Compatible con las tecnologías emergentes

La libertad de elección conlleva a la innovación. Sin embargo, para combinar las arquitecturas de hardware, las plataformas de contenedores y los elementos de microchip adecuados y poder implementar las tecnologías emergentes con eficiencia, se necesita el sistema Linux de nivel empresarial apropiado. Red Hat Enterprise Linux 8 es compatible con diferentes entornos y arquitecturas, lo que le permite tener un sistema operativo estable y uniforme que se adapta a lo largo del proceso de aprendizaje automatico, análisis predictivo, Internet de las cosas (IoT), edge computing y big data.

  • Aprendizaje automático
  • NVIDIA DGX-1 y DGX-2
  • Contenedores para IA

Red Hat Enterprise Linux 8 reduce los obstáculos para adoptar las tecnologías emergentes que admiten las innovaciones de hardware, como las unidades de procesamiento de gráficos (GPU), que pueden utilizarse para las cargas de trabajo de aprendizaje automático. La compatibilidad con las GPU se traduce en más opciones al desarrollar y ejecutar las aplicaciones tradicionales y también en contenedores.

Los servidores de NVIDIA DGX-1 y DGX-2, diseñados para brindar soluciones potentes para los desafíos complejos de inteligencia artificial (IA), están certificados para Red Hat Enterprise Linux. Los sistemas DGX cuentan con capacidades avanzadas para acelerar las cargas de trabajo basadas en GPU. Red Hat Enterprise Linux ha sido optimizado para funcionar en estos sistemas brindando perfiles ajustados personalizados.

Red Hat Enterprise Linux 8 permite implementar y gestionar contenedores de NVIDIA GPU Cloud (NGC) acelerados por GPU en Red Hat OpenShift. Estos contenedores de IA ofrecen pilas y controladores de software integrados para ejecutar marcos de aprendizaje automático optimizados por GPU, como TensorFlow, Caffe2, PyTorch, MXNet, entre otros.

Herramientas de contenedores que le brindan alternativas

Red Hat Enterprise Linux 8 ofrece las herramientas de contenedores que le permiten personalizar los sistemas para buscar, ejecutar, diseñar y compartir contenedores con otras herramientas compatibles con los estándares de la Open Container Initiative (OCI). Con esta mayor variedad de opciones y compatibilidad en cuanto a las aplicaciones en contenedores, usted puede implementar soluciones empresariales cuando y como lo desee.

Intégrese a la comunidad de desarrolladores

  • Podman
  • Buildah
  • Skopeo

Utilice Podman para ejecutar contenedores. Podman es una herramienta de línea de comandos sin daemon que le permite crear y gestionar imágenes en contenedores de forma directa. Al poder trabajar con contenedores sin la necesidad de un entorno de tiempo de ejecución, puede ejercer un control más estricto sobre los permisos otorgados a los elementos de software.

Utilice Buildah para crear contenedores. Buildah es una herramienta de creación que le permite controlar cómo se disponen las capas de imágenes y se accede a los datos durante las creaciones. Para utilizar este programa, no se requiere acceso raíz ni ningún tiempo de ejecución de contenedor. Buildah no incluye herramientas de creación dentro de la imagen misma, lo cual reduce el tamaño de las imágenes que usted diseña.

Utilice Skopeo para buscar y compartir contenedores. Skopeo es una herramienta flexible que permite trasladar, firmar y verificar imágenes entre los servidores de registros y los hosts de contenedores. Utilice Skopeo para examinar imágenes fuera de su sistema

Obtenga el informe técnico para administradores de sistemas

  • Flujos de aplicaciones
  • Consola web
  • Seguridad

Ahora puede ofrecer las aplicaciones que los desarrolladores buscan, independientemente de las versiones del sistema operativo. Utilice flujos de aplicaciones para separar las aplicaciones del sistema operativo base, de forma tal que pueda actualizarlas sin tener que esperar a que salga la próxima versión principal de este.

La nueva consola web brinda una interfaz gráfica basada en explorador para gestionar el sistema Red Hat Enterprise Linux. Con ella, los usuarios de Linux que tienen menos experiencia pueden administrar máquinas virtuales, locales y remotas sin utilizar la línea de comandos. Red Hat Enterprise Linux mejora la gestión de identidades. Los elementos de la gestión de identidades se integran con la consola web para posibilitar un inicio de sesión único (SSO) fácil y la gestión de hosts individuales.

Los controles de seguridad integrados, como las políticas criptográficas de todo el sistema compatibles con OpenSSL 1.1.1 y TLS 1.3, permiten mantener el cumplimiento de la criptografía. Red Hat Enterprise Linux 8 también minimiza la superficie de ataque implementando solamente los paquetes que usted necesita para respaldar su carga de trabajo. Red Hat Enterprise Linux 8 incorpora las mejoras en las políticas de seguridad respecto de los controles de acceso obligatorios de Security-Enhanced Linux (SELinux). De forma proactiva, detecte vulnerabilidades y problemas del sistema relativos a la seguridad, la disponibilidad, el rendimiento y la escalabilidad, y diseñe un plan de corrección, con Red Hat Insights (que ahora se incluye con una suscripción).
